# Проект: Прогнозування опадів на основі даних Open-Meteo

Прототип веб-сервісу для бінарної класифікації погодних умов (наявність або відсутність опадів на основі машинного навчання).

## Структура проєкту:
* `app.py` — головний скрипт застосунку Streamlit, що містить інтерфейс, логіку запитів до API та фінальну ML-модель.
* `weather_daily.csv` — історичний набір даних за 20 років, завантажений через Open-Meteo Historical API(https://open-meteo.com/en/docs?hourly&daily=temperature_2m_min%2Ctemperature_2m_max).
* `requirements.txt` — перелік залежностей та бібліотек для розгортання проєкту.
* `README.md` — документація проєкту.

## Послідовність виконання завдання:
1. **Збір даних:** Запити до Open-Meteo Historical Weather API та Forecast API (агрегація за днями — daily).
   Було обрано дані за період з 2004-01-01 по 2023-12-31. Сайт надає можливість зібрати дані з 1940 року, проте клімат 1940-х років значно відрізняється від клімату 2020-х. Статистичні характеристики (середні температури, частота опадів) змінилися. Якщо модель вчитиметься на даних 1940 року, вона може навчитися закономірностям, які сьогодні вже неактуальні. Також, дані за середину XX століття часто базуються на менш точних інструментах спостереження або моделюванні (реаналіз), ніж сучасні супутникові дані, що може додати "шум" у навчання.
   
2. **Feature Selection:** Відбір 4 інформативних ознак (`temperature_2m_max`, `relative_humidity_2m_mean`, `wind_speed_10m_max`, `pressure_msl_mean`) для усунення мультиколінеарності.
    Для прогнозування дощу або снігу на рівні одного дня природа вимагає конкретних фізичних умов. Наші 4 відібрані ознаки покривають усі фундаментальні метеорологічні стовпи:
   * Вологість (relative_humidity_2m_mean): Головний фактор. Немає вологості — немає хмар і конденсації, а отже, й опадів.
   * Тиск (pressure_msl_mean): Динаміка атмосфери. Падіння тиску сигналізує про прихід циклону (фронту низького тиску), який майже завжди приносить із собою опади.
   * Температура (temperature_2m_max): Енергія. Вона визначає, скільки вологи може утримати повітря та чи перетвориться пара на воду.
   * Вітер (wind_speed_10m_max): Переміщення повітряних мас. Вітер приносить вологі хмари або, навпаки, розганяє їх.
     
     Також можна було включити в навчання моделі не лише максимальнbq поргіг температури 'temperature_2m_max', а й мінімальний 'temperature_2m_min'. Але ці дві ознаки сильно корелювали між собою в correlation matrix(0,95-0,96 в різних періодах часу), що призвело б до мультиколінеарності. З цих двох було обрано максимальний показники тому що метод SelectKBest, який ми запустили, математично побачив, що temperature_2m_max має трохи вищий статистичний зв'язок із опадами, ніж temperature_2m_min
     
3. **Моделювання:** Порівняльний аналіз Логістичної регресії, Дерева рішень та Випадкового лісу.
